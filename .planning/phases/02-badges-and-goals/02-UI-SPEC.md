## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Badge grid gap, compact element spacing |
| md | 16px | Card internal padding, default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Badge icon container: 40px x 40px (rounded-full, centered emoji — between sm and md, matches existing metric tile sizing)
- Badge grid: 4 columns, gap-2 (8px) — matches existing `space-y-2` card pattern
- Touch targets on badge tap: padding ensures 44px minimum tap area — this is an implementation detail handled via padding, not a declared spacing token
- Progress bar height (badge progress): `h-1.5` (6px) is a pre-existing Tailwind class reused from BonusSection.tsx — inherited CSS class reuse, not a new spacing token
- Goal progress bar height: 8px (h-2 — slightly taller than badge progress to signal primary importance)

| Role | Size | Weight | Line Height | Font | Notes |
|------|------|--------|-------------|------|-------|

| Label | 11px (text-[11px]) | 400 (regular) | 1.4 | Fraunces | Badge name under icon in locked state; section sub-labels — uses `font-medium` Tailwind class for visual weight, not a new declared weight |

Tiny label sizes 7px (text-[7px]) and 9px (text-[9px]) are pre-existing project conventions used for metric sublabels — reuse without re-specifying. New Phase 2 content does not introduce sub-7px sizes.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | hsl(var(--background)) — warm cream ~#f8f5f0 light / dark ~#171210 | Page background, badge section background (source: index.css :root) |
| Secondary (30%) | hsl(var(--card)) — pure white light / dark ~#1c1815 | Badge section card, goal area inside HeroCard, badge popup surface |
| Accent (10%) | hsl(var(--primary)) — warm brown ~#8B5E3C light / brighter dark | Reserved for earned badge ring, goal CTA prompt text, on-track goal indicator, confetti primary particle color |
| Destructive | hsl(var(--destructive)) — red ~#dc2626 | Behind-goal status indicator only |

Accent reserved for:
1. Earned badge icon ring/glow (2px solid ring around colorful earned badge)
2. "Set a monthly target ->" prompt text in HeroCard (subtle, tappable)
3. Goal progress bar fill when on-track or ahead
4. Confetti primary burst color (accent + secondary particles)

Semantic coloring for goal status (follows ssColor() pattern — inline style, not Tailwind class):
- Ahead: `#16a34a` (green-600 — same as positive TrendArrow)
- On-track: `hsl(var(--primary))` (warm brown accent)
- Behind: `#dc2626` (red-600 — same as negative TrendArrow / destructive)

Earned badge icons: full color emoji, no filter applied.
Locked badge icons: `opacity-40 grayscale` CSS filter — consistent with "greyed out" per D-02.

## Interaction Contracts

### Badge Grid (BadgeSection)

- Layout: 4-column grid (`grid grid-cols-4 gap-2`) inside the Section card wrapper
- Badge item dimensions: 40px x 40px container, rounded-lg (not rounded-full — matches card radius theme)
- Earned state: full-color emoji, `bg-green-50 dark:bg-green-950/20` background, 2px primary-color ring
- Locked state: emoji with `opacity-40 grayscale` filter, `bg-muted/30` background, no ring
- Tap/click on any badge: shows Radix Tooltip (or small popover) with: badge name (14px), category label (11px muted), progress bar (locked only, h-1.5 reused from BonusSection.tsx), progress text "{n}/{target}" (9px mono), hint text (11px)
- Section header: "Badges ({earned}/{total} earned)" — earned count updates live as data loads
- Section is expandable (uses existing `Section` component wrapper, collapsed state remembered per project pattern)

### Goal Tracker (HeroCard row)

- Position: between Row 2 (metric tiles) and Row 3 (view tabs), separated by a `border-t mt-2 pt-2`
- No goal set state: single-line prompt `Set a monthly target →` in accent color, 11px, tappable full row (padding ensures 44px minimum tap area)
- Goal set state: two-column layout
  - Left: target label "Goal: SS {target}" in 11px medium + status badge chip (Ahead / On track / Behind) with semantic background tint (color + '15' opacity suffix — matches existing chip pattern)
  - Right: current average value in 12px Geist Mono bold + projected trajectory label in 9px muted
- Progress bar: full-width below the two-column row, h-2 (8px), rounded-full, fill color matches goal status semantic color
- Tap/click on goal area (whether set or unset): opens GoalSetDialog

### Goal Set Dialog (GoalSetDialog)

- Uses existing shadcn Dialog (dialog.tsx)
- Dialog title: "Monthly Sleep Score Target" (14px bold)
- Body: Slider component (60–95 range, step 1), default = last month average or 80 if no history
- Slider shows current value as a label above the thumb: "{value}" in 20px bold primary color
- Helper text below slider: "Current month average: {avg}" in 11px muted
- Two actions: "Save Target" (primary button, closes dialog, saves to localStorage) and "Clear Goal" (ghost/text button, destructive color, shown only when goal already set)
- No destructive confirmation for clear — clearing a goal is low-risk and reversible

### Badge Unlock Celebration (Toast + Confetti)

- Toast content: "{badgeIcon} {badgeName} unlocked! +25 XP" — single line, 13px
- Toast duration: 4500ms (per D-09)
- Confetti: CSS-only burst, 12 particles, triggered as sibling div to toast container
- Confetti colors: primary (#8B5E3C equivalent warm brown), amber (#f59e0b), green (#16a34a), and white — palette matches theme warmth
- Confetti particle: 6px x 6px `div`, absolute positioned, `@keyframes confetti-fall` (translate + rotate + fade-out over 1.2s)
- Multiple unlocks: sequential toasts with 500ms delay between each (D-11)
- `prefers-reduced-motion`: confetti div not rendered (check via `window.matchMedia`)

### XP Breakdown Badge Line

- Appears as a new row in XPBreakdown component when badgeXP > 0
- Format: "Badge XP" label (11px) with "+{badgeXP}" value (11px bold, XP_COLOR)
- Consistent with existing breakdown row pattern in XPBreakdown.tsx

## Confetti Animation Spec

New keyframes to add to src/index.css:

```
@keyframes confetti-fall {
  0%   { opacity: 1; transform: translate(0, 0) rotate(0deg); }
  100% { opacity: 0; transform: translate(var(--dx), 60px) rotate(var(--rot)); }
}
```

Each particle is a `div.confetti-particle` with:
- `position: absolute`, `width: 6px`, `height: 6px`, `border-radius: 1px`
- CSS custom properties `--dx` (random -40px to +40px) and `--rot` (random 180deg to 540deg) set via inline style
- Animation: `confetti-fall 1.2s ease-out forwards`
- Stagger: animation-delay 0 to 300ms spread across 12 particles
- Parent: `position: fixed; pointer-events: none; z-index: 9998` (one below noise overlay)

Particle colors (cycle through): `#8B5E3C`, `#f59e0b`, `#16a34a`, `#3b82f6`, `#ffffff` — 5 colors, 12 particles (last 2 repeat first 2 colors).
